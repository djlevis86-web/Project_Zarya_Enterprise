# UI Buttons and Badges Inventory — Project Zarya Enterprise

## 1. Цель

Цель этапа — собрать карту использования кнопок и бейджей перед любыми CSS-правками.

На этом этапе запрещено менять:

- static/css;
- templates;
- бизнес-логику;
- права доступа.

Разрешено только:

- собрать raw-инвентаризацию;
- описать проблемы;
- подготовить безопасный план следующего этапа.

---

## 2. Raw-файлы

Сырые данные сохранены в файлах:

- docs/UI_BUTTONS_BADGES_INVENTORY_RAW.txt;
- docs/UI_BUTTONS_BADGES_CSS_RAW.txt.

---

## 3. Что нужно проанализировать

### 3.1. Кнопки

Нужно найти все варианты:

- btn;
- btn-primary;
- btn-secondary;
- btn-danger;
- btn-sm;
- disabled buttons;
- кнопки в таблицах;
- кнопки в фильтрах;
- кнопки в page-header/action-bar.

### 3.2. Бейджи

Нужно найти все варианты:

- status-badge;
- status-new;
- status-processing;
- status-approved;
- status-rejected;
- status-draft;
- ocr-badge;
- обычные badge-классы.

---

## 4. Главные визуальные проблемы

1. Статусы иногда выглядят как кнопки.
2. Disabled-кнопки занимают место как активные действия.
3. В таблицах кнопки разного размера.
4. Длинные подписи кнопок ломают строки.
5. Бейджи статусов используются в разных контекстах без общей системы.
6. OCR-бейджи и финансовые статусы визуально не отделены друг от друга.

---

## 5. Правильный порядок следующего этапа

feature-ui-buttons-badges-v1

## 6. Что нельзя делать

Нельзя снова делать CSS-заплатки вида:

- добавить новый файл поверх старых;
- переопределить общий класс без анализа;
- менять page-header вместе с кнопками;
- менять таблицы вместе с кнопками;
- менять HTML сразу на нескольких больших страницах без плана.
