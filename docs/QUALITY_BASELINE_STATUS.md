## Покрытая критическая логика

### Доступ к счетам

- список счетов требует авторизации;
- обычный пользователь видит только свои счета;
- staff-пользователь видит все счета;
- обычный пользователь открывает свой счет;
- обычный пользователь не открывает чужой счет;
- staff-пользователь открывает чужой счет.

### Реестр оплаты, service layer

- создается черновик реестра;
- существующий черновик переиспользуется;
- подтвержденный счет добавляется в реестр;
- счет с неподтвержденной OCR-суммой не добавляется;
- активный дубль не добавляется повторно;
- отмененная строка реестра восстанавливается.

### Реестр оплаты, view layer

- добавление в реестр требует авторизации;
- пользователь без права не может добавить счет;
- GET-запрос не добавляет счет;
- staff может добавить подтвержденный счет;
- staff не добавляет неподтвержденный счет;
- строку можно удалить из своего черновика;
- чужую строку удалить нельзя.

## Обязательные проверки перед merge

```bash
python manage.py check
python manage.py makemigrations --check --dry-run
python manage.py migrate --plan
python manage.py test
```
