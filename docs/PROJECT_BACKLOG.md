## 1. Текущий статус

### Готово

- Исправлена форма загрузки счетов: отдельная форма загрузки больше не блокирует файлы из-за лишних обязательных полей.
- Добавлено автоматическое создание OCRJob при отложенном OCR после загрузки.
- Добавлен OCR worker через systemd service/timer.
- Добавлен memory/time guard для OCR:
  - ограничение размера изображения;
  - ограничение количества страниц PDF;
  - timeout Tesseract;
  - ограничение количества Tesseract-вызовов;
  - защита от зависания worker.
- Проблемный файл `агро.pdf` успешно обработался после hotfix.
- Найдена причина плохого OCR по JPG `290626Абакумов*.jpg`: стандартный режим давал мусор, режим `--psm 12` даёт сумму, ИНН и текст.
- Локально создан hotfix `Improve OCR fallback for image invoices`, тесты прошли.

## 5. Правила работы

- Не делать правки наугад.
- Перед исправлением получать фактический вывод команд.
- Работать в отдельной feature/hotfix branch.
- Коммитить только после зелёных тестов.
- `_scripts/` не коммитить.
- Не смешивать hotfix кода и документацию в одном commit.
- После deploy проверять production фактами:
  - git hash;
  - grep нужных строк;
  - `python manage.py check`;
  - systemd status;
  - бизнес-проверка на реальных счетах.
